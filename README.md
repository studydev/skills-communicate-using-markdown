# 마크다운으로 소통하기

_가벼운 텍스트 서식 언어인 마크다운을 사용하여 아이디어를 정리하고 협업하세요._

## 환영합니다

GitHub는 코드 그 이상입니다. GitHub는 소프트웨어 협업을 위한 플랫폼이며, 마크다운은 개발자가 이슈와 풀 리퀘스트에서 명확하고 체계적으로 소통할 수 있게 해주는 가장 중요한 방법 중 하나입니다. 이 실습에서는 제목(heading)을 효과적으로 사용하고, 글머리 기호 목록으로 생각을 정리하고, 체크리스트로 진행 상황을 보여주는 방법을 안내합니다. 이모지, 이미지, 링크를 활용하여 작업에 깊이를 더할 수도 있습니다.

- **대상**: 새로운 개발자, GitHub 초보 사용자, 학생
- **배울 내용**: 마크다운을 사용하여 댓글이나 텍스트 파일에 목록, 이미지, 링크를 추가합니다.
- **만들 것**: 일반 텍스트 파일을 업데이트하고 마크다운 서식을 추가합니다. 이 파일을 사용하여 자신만의 GitHub Pages 사이트를 시작할 수 있습니다.
- **사전 요구사항**: 이 실습에서는 풀 리퀘스트와 파일 편집을 다룹니다. 이러한 기능이 익숙하지 않다면 먼저 [GitHub 소개](https://github.com/skills-kr/introduction-to-github) 실습을 수강하는 것을 권장합니다!
- **소요 시간**: 이 실습은 1시간 이내에 완료할 수 있습니다.

이 실습에서 다음을 수행합니다:

1. 헤딩 추가하기
2. 이미지 추가하기
3. 코드 예제 추가하기
4. 작업 목록 만들기
5. 풀 리퀘스트 병합하기

2. 저장소에 생성된 이슈의 단계별 안내를 따르세요.
3. 페이지가 자동으로 새로고침되지 않으면 [Actions](../../actions) 탭을 확인하세요.

   - 실패한 작업이 표시되면 이슈를 제출해 주세요. 버그를 발견하셨네요! 🐛
